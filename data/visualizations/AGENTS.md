# Visualizations Technical Reference

## Overview

Technical documentation for visualization file formats and naming conventions.

## File Types

### PNG Charts

**Format**: PNG image files

**Entity-Specific Charts** (no timestamps):
- **Naming**: `{entity}_{type}.png`
- **Types**:
  - `complexity_analysis`: Complexity analysis charts
  - `learning_objectives`: Learning objectives visualization
  - `section_breakdown`: Section breakdown charts
  - `technical_content`: Technical content analysis
- **Examples**:
  - `Barry Bonds_complexity_analysis.png`
  - `Coffee Roasting_learning_objectives.png`

**Global Aggregate Charts** (no timestamps):
- **Naming**: `curriculum_metrics.png`
- **Purpose**: Aggregated metrics dashboard for all curricula

### Mermaid Diagrams

**Format**: Mermaid diagram files (.mmd)

**Entity-Specific Diagrams** (no timestamps):
- **Naming**: `{entity}_flow.mmd`
- **Purpose**: Individual learning flow diagrams for each entity/domain
- **Examples**: `Barry Bonds_flow.mmd`, `Coffee Roasting_flow.mmd`

**Global Structure Diagram** (no timestamps):
- **Naming**: `curriculum_structure.mmd`
- **Purpose**: Overall curriculum structure diagram for all curricula

### JSON Metrics

**Format**: JSON data files
**Naming**: `curriculum_metrics.json`

**Schema**: Array of curriculum metadata objects:
```json
[
  {
    "entity_name": "string",
    "word_count": number,
    "paragraph_count": number,
    "section_count": number,
    "sections": ["section1", "section2", ...],
    "objectives_count": number,
    "code_block_count": number,
    "math_expressions_count": number,
    "words_per_section": number,
    "words_per_paragraph": number
  }
]
```

**Note**: This file is generated by the visualization script and contains detailed metrics for all curricula.

## Generation

Files are generated by:
- `learning/curriculum_creation/3_Introduction_Visualizations.py` - Creates JSON metrics, PNG charts, and Mermaid diagrams
- `src/visualization/runner.py::generate_curriculum_metrics()` - Creates CSV metrics (saved to `metrics/` subdirectory)
